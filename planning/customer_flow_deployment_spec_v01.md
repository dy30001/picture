# 客户选片与部署门禁规格 v01

Date: 2026-05-02
Mode: doc
Workspace: `/Users/dy3000/code/pic`

## 1. 客户视角目标

客户不需要理解模型、模板库或内部文件夹，只需要按步骤完成选择、确认和交付。

- 好选：先选业务场景，再选人物关系、风格、张数和交付方式。
- 清晰：页面只展示当前步骤、当前选择、下一步和必要证据。
- 可用：每一步上线到服务器前必须有健康检查、接口检查和页面检查。
- 严谨：三视图确认是正式样片和批量成片的前置门禁。

## 2. 主流程

### 2.1 选择场景

客户首先选择一个主场景，不直接进入 2300+ 模板库。

首批主场景：

- 婚纱照
- 情侣照
- 闺蜜照
- 儿童 10 岁照
- 女生写真
- 夕阳红

界面要求：

- 每个场景只显示名称、适合人群、推荐张数和一个主按钮。
- 默认推荐排在前面，历史/更多模板默认折叠。
- 不在首屏展示内部模板 ID、模型名、长提示词。

### 2.2 上传与身份确认

客户上传参考图后，系统进入身份基准流程。

步骤：

1. 上传参考图。
2. 生成或选择三视图基准。
3. 客户确认“像本人”。
4. 三视图通过后，才开放模板样片。

状态口径：

- `待上传`：还没有参考图。
- `待三视图`：已上传，未生成或未选择三视图。
- `待确认`：三视图已出，需要客户确认。
- `已确认`：可进入模板样片。
- `需返修`：身份不像、比例不对或客户拒绝。

界面要求：

- 主界面只显示当前身份状态。
- 三视图证据用缩略图展示，详细记录折叠。
- 不允许在未确认三视图时展示“批量生成成片”的主动作。

### 2.3 模板样片

模板样片用于让客户确定风格，不承担最终交付。

步骤：

1. 系统按场景给出 3 到 6 个推荐模板。
2. 客户选择 1 到 3 个方向。
3. 生成小批样片。
4. 客户确认风格或要求返修。

界面要求：

- 推荐模板优先，完整模板库放到“更多模板”。
- 每个模板卡片只保留：标题、适用场景、效果关键词、使用按钮。
- 长提示词只在详情里展示。
- 样片区与最终成片区分开，避免客户误以为样片就是交付。

### 2.4 批量成片

批量成片只在身份和样片都通过后启动。

步骤：

1. 确认场景、数量、横竖版和交付规格。
2. 启动批量生成。
3. 生成中展示当前批次进度。
4. 生成后进入复核。

状态口径：

- `排队中`
- `生成中`
- `待复核`
- `需返修`
- `可交付`

界面要求：

- 进度只保留一个单一来源。
- 列表紧凑展示：编号、场景、状态、预览、操作。
- 失败原因使用客户能理解的口径，例如“脸不像”“画面裁切不完整”“清晰度不足”。

### 2.5 交付

交付页只展示最终可用结果。

内容：

- 最终图片
- 下载入口
- 返修记录
- 验收状态
- 交付数量核对

界面要求：

- 样片、失败图、被替换图默认不进入交付页。
- 交付页只保留一个主动作：下载或确认交付。
- 详细日志放在折叠证据区。

## 3. 页面信息架构

### 3.1 首页

首页只回答三个问题：

- 现在做什么场景？
- 当前流程到哪一步？
- 下一步点哪里？

推荐布局：

1. 顶部：产品名、当前项目、服务状态。
2. 主流程条：场景选择、身份确认、模板样片、批量成片、交付。
3. 当前步骤面板：只展示本步骤需要的操作。
4. 证据区：默认折叠，展示健康检查、生成记录、接口状态。

### 3.2 场景选择页

推荐用紧凑卡片，不使用大面积营销式介绍。

每张卡片字段：

- 场景名称
- 推荐人群
- 推荐张数
- 当前可用状态
- 主按钮

### 3.3 身份确认页

核心内容：

- 上传参考图
- 三视图预览
- 确认/返修按钮
- 当前状态

不展示：

- 内部 prompt 调试过程
- 模型试错过程
- 过长身份描述

### 3.4 样片与成片页

样片页用于选风格，成片页用于交付。

必须分开：

- 样片：可比较、可返修、可选择方向。
- 成片：可复核、可下载、可交付。

### 3.5 设置与高级项

默认折叠：

- API URL
- API Key
- 接口模式
- 模型名
- 超时时间
- 完整模板库

客户主流程不依赖客户理解这些字段。

## 4. 服务器部署门禁

每一步部署到服务器前，必须按以下顺序检查。

### 4.1 启动前检查

- 确认工作区：`/Users/dy3000/code/pic`
- 确认依赖完整：`npm install` 已完成或 `node_modules` 存在。
- 确认端口规划：
  - 通用图像工作台：`4174`
  - 成片选片复核台：`4184`
- 确认监听地址：
  - 本机验证可用 `127.0.0.1`
  - 局域网或服务器访问必须使用 `0.0.0.0` 或服务器实际 IP

### 4.2 启动命令

通用图像工作台：

```bash
node server/index.mjs --host 0.0.0.0 --port 4174
```

成片选片复核台：

```bash
node identity-workflow/server.mjs --host 0.0.0.0 --port 4184
```

### 4.3 健康检查

通用图像工作台：

```bash
curl -fsS http://127.0.0.1:4174/api/health
curl -fsS http://SERVER_IP:4174/api/health
```

成片选片复核台：

```bash
curl -fsS http://127.0.0.1:4184/api/health
curl -fsS http://SERVER_IP:4184/api/health
```

通过标准：

- HTTP 状态为 `200`
- JSON 中 `ok` 为 `true`
- 服务名与目标程序一致

### 4.4 接口检查

通用图像工作台必须检查：

- `/api/templates`
- `/api/templates/portrait-axis-female-3view-v01`
- `/api/templates/portrait-best-friends-studio-v01`
- `/api/history`
- `/api/test-connection`

成片选片复核台必须检查：

- `/api/workflow/status`
- `/api/generation/jobs`
- `/api/auth/options`

通过标准：

- 模板接口能返回本地人像模板。
- 三视图模板可读。
- 闺蜜照模板可读。
- 工作流状态能返回身份基准、批次和交付数量。
- 认证选项接口可读，不暴露密码或敏感信息。

### 4.5 页面检查

桌面端：

- 首页首屏能看见当前主流程。
- 主动作清楚。
- 模板卡片不撑破布局。
- 状态不重复。

移动端：

- 无横向滚动。
- 按钮文字不挤压。
- 卡片不超宽。
- 当前步骤仍在首屏可识别。

### 4.6 生成链路检查

正式交付前至少检查：

- API 设置保存后不泄露 Key。
- 测试连接能返回明确成功或失败。
- 上传参考图后页面有预览。
- 生成失败时能显示可读错误。
- 历史记录能写入、读取、删除和恢复。

### 4.7 回滚标准

出现以下情况必须暂停部署并回滚：

- 健康检查失败。
- 页面可打开但核心接口失败。
- 模板样片和成片混在同一交付区。
- 未确认三视图却能启动批量成片。
- API Key 或敏感配置出现在历史、状态栏或截图里。
- 移动端出现横向滚动或主动作不可见。

## 5. 验收清单

客户流程验收：

- 客户能从场景入口开始，不需要搜索模板 ID。
- 客户能明确知道当前步骤和下一步。
- 三视图确认前不能进入批量成片。
- 样片和成片在界面上明确分开。
- 交付页只展示最终可用结果。

部署验收：

- 本机健康检查通过。
- 服务器或局域网健康检查通过。
- 模板接口通过。
- 工作流接口通过。
- 桌面和移动视觉检查通过。
- 敏感信息没有出现在页面状态、历史或截图中。

## 6. 后续 code 模式建议

下一轮如果进入代码执行，建议拆成三步：

1. 客户流程入口：把场景选择、当前步骤和下一步收敛到首页。
2. 身份门禁：未确认三视图时隐藏或禁用批量成片主动作。
3. 部署验证：补一条脚本化检查，把健康、模板、页面和视觉检查串起来。
