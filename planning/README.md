# planning 文档目录

`planning/` 现在按“主线规范”和“过程文档”分层，不再把品牌、架构、流程、计划、报告、CSV 都堆在同一层。

## 1. 目录结构

```text
planning/
  README.md
  01-架构与规范/      品牌、架构、模块和统一入口口径
  02-产品与交付/      产品设计、客户流程、样片和验收规范
  03-流程与提示词/    身份锁定、工作流、提示词和服装矩阵
  04-研究与交接/      专项研究、说明和线程交接
  05-执行计划/        单次任务计划，默认本地过程文档
  06-执行报告/        单次任务报告，默认本地过程文档
  07-结构化清单/      CSV 清单、镜头表、队列表，默认本地过程文档
```

## 3. 分类说明

### `01-架构与规范`

- 放品牌规范、开发板块总图、模块实施规范、代码结构规划、统一入口现状等长期口径。

### `02-产品与交付`

- 放产品设计、客户流程、样片管理、质量验收等面向产品和交付的正式文档。
- 新增客户语言规范时，优先放这里，再同步到 `docs/README.md` 和根 `README.md`。

### `03-流程与提示词`

- 放人脸锁定、身份门禁、婚纱生成工作流、提示词系统、风格矩阵和服装分配策略。

### `04-研究与交接`

- 放专项研究、背景说明、线程交接和只需长期保留的上下文文档。

### `05-执行计划`

- 放 `主题-执行计划.md` 单次任务计划。
- 如需区分同主题变体，用 `主题-补充说明-执行计划.md`。
- 不再使用 `v01`、`v02` 这类通用版本尾巴。
- 默认视为过程文档，不作为主入口。

### `06-执行报告`

- 放 `主题-执行报告.md` 历史执行证据。
- 批次型执行记录统一用 `主题-YYYYMMDDHHMMSS-执行报告.md`。
- 不再拼接 `kes1`、`57uf` 这类随机尾码。
- 可以追脚本、产物、命令，但不代替正式规范。

### `07-结构化清单`

- 放 `主题.csv` 清单、镜头表、服装分配表和生成队列。
- 如确实需要多版并存，用 `主题-第一版.csv`、`主题-第二版.csv`。

## 5. 维护规则

- 新增长期规范时，只能放到 `01-04` 对应分类，不再回到 `planning/` 根目录。
- 新增一次性计划、执行报告、CSV 清单时，分别放到 `05`、`06`、`07`。
- 新增命名时优先中文主题，不混用无意义英文缩写、通用 `v01/v02` 和随机尾码。
- 提示词正文、命令、接口字段和代码路径允许保留英文，但标题、索引和结论统一写中文。
- 如果某次执行报告沉淀出长期规则，要把结论提炼回 `01-04` 的正式文档。
- 新增或改名后，同步更新本文件、`docs/README.md` 和需要暴露的根 `README.md`。
