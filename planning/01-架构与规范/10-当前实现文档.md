## 0. 一句话结论

当前实现是一套 Node 主应用：`server/index.mjs --port 9999` 提供客户前台、模板、生图、历史、账号、积分、支付、样片和定制订单接口；客户前端在 `public/`，正式成片目录只认 `final_4k/`。

本文档记录当前实现事实，不替代产品设计；产品口径以 `../02-产品与交付/03-个人AI摄影棚产品设计.md` 为准。

## 1. 当前运行入口

当前脚本口径：

| 命令 | 实际入口 |
| --- | --- |
| `npm run dev` | `node --watch server/index.mjs --host 127.0.0.1 --port 9999` |
| `npm start` | `node server/index.mjs --host 127.0.0.1 --port 9999` |
| `npm run preview` | `node server/index.mjs --host 127.0.0.1 --port 9999` |
| `npm run open` | `scripts/open-workbench.mjs` |

健康检查：

```bash
curl -fsS http://127.0.0.1:9999/api/health
```

客户前台不展示内部端口号；`9999` 是部署和调试口径。

## 3. 主服务接口

### 3.1 基础与样片

| 接口 | 用途 |
| --- | --- |
| `GET /api/health` | 服务健康检查 |
| `GET /api/studio-samples` | 摄影棚样片目录，默认轻量返回 |
| `GET /api/studio-samples/:sceneId/:groupId` | 单个样片组详情 |
| `GET /studio-previews/*` | `final_4k/` 正式成片静态预览 |
| `GET /studio-review/*` | `review/` 复核证据静态资源 |

### 3.2 定制订单

| 接口 | 用途 |
| --- | --- |
| `GET /api/studio-orders` | 当前客户的定制订单列表 |
| `POST /api/studio-orders` | 当前客户创建定制订单 |
| `GET /api/admin/studio-orders` | 后台查看订单列表 |
| `POST /api/admin/studio-orders/:id` | 后台更新订单 |

客户订单接口要求账号会话；后台订单接口要求后台密钥。后台接口是内部实现入口，不等于客户前台可见页面。

### 3.3 账号

| 接口 | 用途 |
| --- | --- |
| `GET /api/auth/options` | 当前登录方式 |
| `POST /api/auth/verification-code` | 发送验证码 |
| `POST /api/auth/register` | 创建账号 |
| `POST /api/auth/password-reset` | 重置密码 |
| `POST /api/auth/login` | 登录 |

当前客户账号口径是邮箱注册和登录；客户账号不等同于后台管理员账号。

### 3.4 模板与生图

| 接口 | 用途 |
| --- | --- |
| `GET /api/templates` | 模板轻量列表 |
| `GET /api/templates/:id` | 模板详情 |
| `POST /api/test-connection` | 内部连接测试 |
| `POST /api/generate` | 图片生成或编辑 |

`/api/test-connection` 是内部调试接口，不作为客户前台显式入口。

### 3.5 积分和支付

| 接口 | 用途 |
| --- | --- |
| `GET /api/credits` | 积分账户 |
| `POST /api/credits/estimate` | 生成前积分估算 |
| `POST /api/credits/recharge` | 本地模拟充值或直接充值入口 |
| `GET /api/credits/orders` | 充值订单列表 |
| `POST /api/credits/orders` | 创建充值订单 |
| `GET /api/payments/config` | 支付配置 |
| `POST /api/payments/checkout-session` | 创建支付会话 |
| `POST /api/payments/confirm-session` | 确认支付会话 |
| `POST /api/payments/stripe/webhook` | Stripe 回调 |
| `POST /api/payments/alipay/notify` | 支付宝异步通知 |
| `GET /api/payments/alipay/return` | 支付宝返回页 |

客户界面只展示 `我的积分`、`买积分`、`充值记录`、`积分明细` 等客户词。

### 3.6 历史

| 接口 | 用途 |
| --- | --- |
| `GET /api/history` | 作品历史 |
| `POST /api/history/sync` | 同步历史 |
| `DELETE /api/history/:id` | 软删除 |
| `POST /api/history/:id/restore` | 恢复 |
| `DELETE /api/history/:id/permanent` | 永久删除单条 |
| `DELETE /api/history` | 清空已删除 |

历史记录用于普通图片创作，不替代正式定制成片交付目录。

## 4. 存储实现

当前存储有两种模式：

| 模式 | 触发条件 | 说明 |
| --- | --- | --- |
| 文件模式 | 未启用 PostgreSQL | 账号、积分、订单、历史等落在 `data/` 下 |
| PostgreSQL 模式 | `APP_STORAGE_BACKEND=postgres` 或 `STORAGE_BACKEND=postgres` | 账号、积分、订单、历史走 PostgreSQL |

PostgreSQL 主要配置：

- `POSTGRES_HOST`
- `POSTGRES_PORT`
- `POSTGRES_DATABASE`
- `POSTGRES_USER`
- `POSTGRES_PASSWORD`

初始化和迁移脚本：

```bash
npm run db:init
npm run db:migrate:file-to-postgres
```

当前还有英文库到中文库迁移脚本：

```bash
npm run db:migrate:english-to-chinese
```

使用 PostgreSQL 前必须先保证数据库、角色和连接配置完整。

## 5. 业务产物目录

| 目录 | 当前口径 |
| --- | --- |
| `generated/` | 生成过程源图和中间结果 |
| `review/` | 联系表、修复队列、复核证据 |
| `final_4k/` | 唯一正式成片目录 |
| `成片/` | 历史重复目录，只保留兼容数据 |
| `data/` | 本机历史、账号、积分、订单和日志 |
| `public/template-previews/` | 模板和样片本地预览图 |

客户成片页只能展示 `final_4k/` 里的正式结果；过程稿、失败图和工作人员备注不进入客户结果页。

## 6. 当前客户前台

客户前台 6 个业务板块：

1. `首页`
2. `影像定制`
3. `图片创作`
4. `我的作品`
5. `登录`
6. `我的积分`

客户前台不展示：

- `测试连接`
- `后台制作平台`
- `三视图`
- `人物确认图`
- `像不像审核`
- `后台上传`
- 模型参数和内部端口

## 7. 当前后台实现边界

当前服务端已经有后台订单接口：

```text
GET /api/admin/studio-orders
POST /api/admin/studio-orders/:id
```

这些接口通过 `INKLENS_STUDIO_ADMIN_KEY` 或 `STUDIO_ADMIN_KEY` 做后台门禁。当前默认本地开发密钥是 `mojing-admin-local`。

注意：

- 这只是后台实现入口和接口门禁，不代表后台平台已经并入客户前台。
- 后台制作平台仍按独立内网端口和管理员登录方向规划。
- 客户前台不能出现后台入口。

## 9. 剩余实现风险

| 风险 | 当前判断 | 后续要求 |
| --- | --- | --- |
| 后台平台未完全独立成页面 | 服务端已有后台订单接口，客户前台不展示后台入口 | 后续单独规划内网后台页面和登录 |
| `成片/` 历史目录仍存在 | 不再作为正式交付口径 | 后续做目录清理和迁移核查 |
| `public/ + server/` 仍是当前实现根 | `app/` 只是目标结构说明 | 迁移前继续按当前目录维护 |
| 支付真实到账 | 接口链路和回调存在 | 真实支付需单独做安全验证 |
| 图片资产数量大 | Git 发布和图片同步应分开 | 按香港公网更新文档执行 |
