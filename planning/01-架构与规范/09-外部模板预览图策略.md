# 外部模板预览图策略

日期：2026-05-03  
模式：doc  
工作区：`/Users/dy3000/code/pic`

## 1. 口径先定清楚

### 1.1 什么才叫模板

- `图片创作 -> 模板库` 里的第三方模板才叫模板。
- 当前外部模板主要来自两层：
  - `public/README_zh.md`
  - `public/sorry-templates.json` 里的外部导入部分

### 1.2 什么不叫模板

- 当前那 `17` 个本地项，不算模板。
- 它们只是摄影棚流程里的本地样片、封面素材、场景参考资产。
- 即使这些本地项暂时和 `/api/templates` 走在同一条代码路径里，产品口径上也不能再把它们叫“模板”。

## 2. 当前图片链路

### 2.1 外部模板图现在怎么来

1. `scripts/import-sorry-templates.mjs` 从外部模板源读取 `preview_url`
2. `public/sorry-templates.json` 保存外部模板的 `imageUrl`
3. 前端 `public/app.js` 的 `templateRemoteImage()` 对 `cms-assets.youmind.com` 再套一层 `wsrv.nl`
4. 浏览器实际加载的是：
   - 外部图源 `cms-assets.youmind.com`
   - 再经过第三方代理 `wsrv.nl`

### 2.2 占位图什么时候出现

- `imageUrl` 为空时出现占位图
- 远程图加载失败时出现占位图
- 这件事由 `public/app.js` 的 `hydrateTemplateImages()` 控制

## 3. 这次核查结论

### 3.1 慢的不是本地项，是外部模板图链路

- 本地那 `17` 个项不属于这次问题定义
- 当前“很多卡片看起来像占位符”的核心问题，是外部模板图加载慢或失败

### 3.2 外部模板图应该本地化

- 结论：**应该下载下来，放到我们自己的服务器上**
- 但优先下载的是**模板预览图/封面图**，不是先追求所有原始大图

## 4. 为什么不是先传原始大图

当前模板库主界面只需要列表封面，不需要原始大图做放大查看。

所以第一步应该是：

1. 下载外部模板预览图
2. 存到本地静态目录或我们自己的对象存储
3. 模板 JSON 改为优先指向本地预览图
4. 保留原始外链作为来源元数据，不作为前台首选加载地址

不建议第一步就做：

- 为全部模板追原始高清大图
- 继续依赖 `cms-assets.youmind.com -> wsrv.nl` 双外部链路

## 5. 推荐实施顺序

### 阶段 A：先把模板卡片变稳

- 给每个外部模板生成稳定的本地预览图路径
- 前台模板卡片优先使用本地预览图
- 远程地址只作补充元数据，不作首屏依赖

### 阶段 B：再决定要不要保留原始图

只有在下面情况出现时，才值得补“原始图下载”：

- 模板详情页要看大图
- 需要离线归档
- 需要避免第三方源后续失效

## 6. 以后默认说法

- 说“外部模板”时，指第三方模板库条目
- 说“本地样片/封面素材”时，指那 `17` 个本地项和摄影棚参考资产
- 不再把那 `17` 个本地项叫模板
