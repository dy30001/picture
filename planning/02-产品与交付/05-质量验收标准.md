## 7. 评分标准

### 7.1 总分规则

- 单张满分：100 分。
- 通过线：85 分及以上，且不存在 P0 或 P1 问题。
- 条件通过：80-84 分，且仅存在 P2/P3 问题；需记录问题，进入批次复查。
- 返修：低于 80 分，或存在任一 P0/P1 问题。

### 7.2 评分维度

| 维度 | 分值 | 验收要点 | 扣分参考 |
| --- | ---: | --- | --- |
| 分辨率与清晰度 | 15 | 输出为 4K，主体清晰，边缘无明显糊化，细节足够支撑成片交付 | 非 4K、主体虚焦、过度锐化、局部像素破碎 |
| 人物结构 | 20 | 面部、手指、肢体、肩颈、躯干比例自然，无多肢、多指、错位或融合 | 手部异常、五官崩坏、肢体畸形、人物互相粘连 |
| 婚纱与造型 | 15 | 婚纱材质、裙摆、头纱、妆发、配饰稳定，符合婚纱主题 | 服装断裂、头纱穿模、饰品漂浮、材质塑料感强 |
| 构图与镜头意图 | 15 | 符合镜头清单意图，主体位置合理，留白克制，画面重心稳定 | 构图偏离、主体被裁切、背景抢戏、视角不符合要求 |
| 光线与色彩 | 10 | 光线方向一致，肤色自然，白纱不过曝，整体色调统一 | 过曝、脏灰、肤色偏色、光影冲突 |
| 场景一致性 | 10 | 场景、季节、时间、道具和人物关系符合设定 | 背景错误、道具不合主题、空间透视混乱 |
| 商业可用性 | 10 | 无水印、无乱码文字、无侵权标识，适合交付和展示 | 随机文字、Logo、水印、明显平台痕迹 |
| 文件规范 | 5 | 文件命名、格式、版本号、目录位置符合约定 | 命名错误、编号缺失、版本混乱、格式不符 |

## 8. 批次验收流程

### 8.1 批次划分

- 建议按 9 张一批，共 9 批。
- 每批先做快速筛查，再做逐张评分。
- 每批验收完成后再进入下一批，避免问题模式重复扩散。

### 8.2 单批流程

1. 文件完整性检查
   - 确认本批编号连续。
   - 确认每张图均为 4K 输出。
   - 确认无空文件、重复文件或错误格式。

2. 快速视觉筛查
   - 先标记明显失败图：人物崩坏、主体错误、非婚纱主题、严重糊图。
   - 明显失败图直接进入返修队列，不占用细评分时间。

3. 逐张评分
   - 按 8 个维度打分。
   - 每张图记录总分、失败类型、返修优先级和备注。
   - 对 80-84 分图片标记为条件通过，等待批次复查。

4. 批次复查
   - 检查同批是否存在重复构图、重复脸、重复背景或同类瑕疵。
   - 若同类问题超过 3 张，回看该批生成参数或提示词方向。

5. 批次结论
   - 通过：本批 9 张全部达到通过线。
   - 条件通过：本批无 P0/P1，但存在少量 P2/P3，已记录。
   - 返修：任一 P0/P1 未修复，或通过数低于 8 张。

## 9. 常见失败类型

| 类型 | 说明 | 默认优先级 |
| --- | --- | --- |
| 人物手部异常 | 多指、少指、手指融合、手掌结构错误 | P1 |
| 面部崩坏 | 五官错位、眼睛不一致、表情僵硬或恐怖谷 | P1 |
| 肢体结构错误 | 多手臂、多腿、肩颈断裂、躯干比例异常 | P0 |
| 人物关系错误 | 新郎新娘数量错误、站位关系错误、拥抱或牵手穿模 | P0 |
| 婚纱错误 | 不是婚纱、婚纱严重破碎、裙摆穿模 | P1 |
| 主体被裁切 | 头部、手部、裙摆或关键人物被不合理裁掉 | P1 |
| 构图偏离 | 不符合镜头清单中的景别、动作或氛围 | P1 |
| 清晰度不足 | 主体虚焦、局部糊成块、细节无法商用 | P1 |
| 光线冲突 | 人物和背景光源方向明显不一致 | P2 |
| 色彩脏灰 | 白纱发灰、肤色偏绿偏红、整体不通透 | P2 |
| 背景异常 | 建筑变形、地平线错乱、透视不稳定 | P2 |
| 随机文字或水印 | 出现乱码、签名、水印、品牌标识 | P0 |
| 重复度过高 | 多张脸、构图、背景高度相似 | P2 |
| 文件不规范 | 编号、版本、格式或目录不符合约定 | P3 |

## 10. 返修优先级

### P0：必须立即返修

- 人物数量或人物关系错误。
- 肢体严重畸形，影响基本观感。
- 出现水印、乱码文字、品牌标识或不可交付元素。
- 输出不是 4K，或文件损坏、无法打开。

处理要求：
- 不进入最终候选。
- 优先重新生成或重做局部修复。
- 修复后必须重新完整评分。

### P1：高优先级返修

- 面部、手部、婚纱、主体裁切、镜头意图存在明显问题。
- 单张图低于 80 分。
- 画面主题明显偏离婚纱交付目标。

处理要求：
- 进入当批返修队列。
- 修复后复查相关维度，并确认无新增问题。

### P2：中优先级优化

- 色彩、光线、背景细节、轻微重复度问题。
- 不影响基本交付，但会影响高级质感。

处理要求：
- 可随批次统一调色、重裁或局部修复。
- 若同类问题集中出现，应回溯生成策略。

### P3：低优先级整理

- 文件命名、备注、目录、版本号等管理问题。
- 不影响图片画面，但影响交接效率。

处理要求：
- 最终交付前统一整理。
- 不因 P3 单独阻塞视觉验收。

## 11. 文件命名规范

### 11.1 原则

- 编号固定，避免后续替换时错位。
- 版本清晰，保留返修记录。
- 文件名只使用 ASCII 字符、数字、下划线和短横线。

### 11.2 命名格式

```text
wedding_###_scene-key_v##_status.ext
```

字段说明：

- `wedding`：项目固定前缀。
- `###`：三位镜头编号，范围 `001` 到 `081`。
- `scene-key`：简短场景关键词，例如 `garden-wide`、`veil-closeup`。
- `v##`：两位版本号，例如 `v01`、`v02`。
- `status`：状态，可选 `draft`、`review`、`fix`、`final`。
- `ext`：文件扩展名，最终建议使用 `png` 或高质量 `jpg`。

示例：

```text
wedding_001_garden-wide_v01_review.png
wedding_001_garden-wide_v02_final.png
```

### 11.3 编号规则

- 镜头编号必须与 81 张清单编号一致。
- 返修不改变镜头编号，只递增版本号。
- 最终交付文件每个编号只保留一个 `final` 版本。
- 废弃文件不得覆盖最终文件，应移入归档或保留为非交付版本。
