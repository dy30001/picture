# 客户流程与部署规范

日期： 2026-05-02
模式： doc
工作区： `/Users/dy3000/code/pic`

## 0. 当前总口径

1. 客户统一访问 `http://<host>:9999`
2. 当前正式运行服务是 `server/index.mjs`
3. 正式成片目录是 `final_4k/`
4. 样片和成片必须分开
5. `成片/` 只保留历史兼容数据，不再作为正式交付目录
6. 批量成片允许直接生成 4K，批次复核默认直接基于 4K 结果进行

## 1. 客户主功能

客户不需要理解模型、模板库或内部文件夹，只需要先看样片集，再按生图流程完成生成。

当前主功能固定拆成两块：

1. `样片集`
2. `生图`

### 1.1 样片集

样片集是独立功能，用于展示成片效果和可参考的拍摄方向。

样片集只回答：

- 有哪些好看的样片？
- 每组样片适合什么场景？
- 用户想参考哪个方向？

样片集不承担：

- 上传参考照
- 身份确认
- 正式生成
- 交付管理

### 1.2 生图流程

生图流程固定为：

1. 传三张图
2. 确认像本人
3. 选模板作为生成参考
4. 正式生成

硬规则：

- 未传满三张图，不进入身份确认。
- 未完成身份确认，不进入模板选择。
- 未选择模板，不开放正式生成主动作。

## 2. 页面要求

### 2.1 首页

首页只回答两个问题：

- 先看哪个样片方向
- 是否进入生图流程

推荐布局：

1. 顶部：产品名、当前项目、服务状态
2. 样片集：大图样片、场景分类、风格标签
3. 生图入口：传三张图、身份确认、选模板、生成
4. 证据区：默认折叠，展示健康检查、生成记录、接口状态

### 2.2 身份确认页

必须做到：

- 上传三张参考图
- 展示身份确认预览
- 提供确认/返修动作
- 未确认前不开放正式批量成片主动作

### 2.3 样片集

样片集用于看效果，不承担最终交付。

必须做到：

- 大图优先
- 分类清楚
- 能快速切换婚纱照、情侣照、闺蜜照、10 岁照、女生写真、夕阳红
- 样片集和正式生图区分开

### 2.4 模板与生成页

模板用于提供生成参考。

必须做到：

- 模板选择发生在身份确认之后
- 选中模板后回到生成步骤
- 模板卡片只在模板库或模板选择区展示
- 生图页只展示最终提示词、参数和生成动作

### 2.5 成片与交付页

交付页只展示正式可用结果。

必须做到：

- 正式图片来自 `final_4k/`
- 样片、失败图、被替换旧图默认不进交付页
- 交付页只保留一个主动作：下载或确认交付

## 3. 状态门禁

### 3.1 身份门禁

生图流程只认以下状态：

- `待上传`
- `待生成`
- `待确认`
- `已确认`
- `需返修`

硬规则：

- 未传满三张图前，不能进入身份确认。
- 未完成身份确认前，不能进入模板选择。
- 未选择模板前，不能启动正式生成。

### 3.2 成片门禁

批量成片只在身份和样片都通过后启动。

当前技术口径：

- 正式批量阶段允许直接输出 4K 成片候选
- 批次复核直接看 4K 结果
- `final_4k/` 负责承接复核通过后的正式交付版本

状态建议：

- `排队中`
- `生成中`
- `待复核`
- `需返修`
- `可交付`

## 4. 部署门禁

### 4.1 启动前检查

- 工作区：`/Users/dy3000/code/pic`
- 依赖完整：`npm install` 已完成或 `node_modules` 存在
- 当前正式端口：`9999`
- 当前正式服务：`server/index.mjs`
- 当前正式成片目录：`final_4k/`

### 4.2 启动命令

```bash
node server/index.mjs --host 0.0.0.0 --port 9999
```

说明：

- 这是当前唯一正式启动命令
- 旧的 `identity-workflow` 独立启动方式不再作为现行部署口径

### 4.3 健康检查

```bash
curl -fsS http://127.0.0.1:9999/api/health
curl -fsS http://SERVER_IP:9999/api/health
```

通过标准：

- 返回 `ok: true`
- 服务名正确
- 页面可正常加载主界面

### 4.4 页面检查

部署前至少检查：

1. 首页能打开
2. 样片集默认可见，并且只展示样片、场景和风格方向
3. 生图流程可见，并且只按传三张图、身份确认、选模板、生成推进
4. 没有页面级横向滚动
5. 按钮、标题、卡片不松散

## 5. 目录与交付门禁

### 5.1 正式交付目录

正式交付只认：

```text
final_4k/
```

### 5.2 历史兼容目录

历史重复目录是：

```text
成片/
```

它当前只允许：

- 保留历史数据
- 作为清理前的兼容痕迹

它当前不允许：

- 再作为正式交付目录
- 再作为脚本默认同步目标
- 再作为正式文档里的交付口径
