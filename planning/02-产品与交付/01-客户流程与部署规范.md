# 客户流程与部署规范

日期： 2026-05-02
模式： doc
工作区： `/Users/dy3000/code/pic`

## 0. 当前总口径

1. 客户统一访问 `http://<host>:9999`
2. 当前正式运行服务是 `server/index.mjs`
3. 正式成片目录是 `final_4k/`
4. 样片和成片必须分开
5. `成片/` 只保留历史兼容数据，不再作为正式交付目录
6. 批量成片允许直接生成 4K，批次复核默认直接基于 4K 结果进行
7. 影像定制按“一个订单，两套平台”执行：客户平台面向注册用户，后台制作平台面向管理员和工作人员
8. 客户只关注结果；`三视图`、`人物确认图`、`像不像审核`、`后台上传` 都是内部生产动作，不进入客户主流程

## 1. 客户主功能

客户不需要理解模型、内部文件夹或后台制作，只需要先看样片集，再提交资料、等待制作、查看成片。

当前客户平台固定拆成四块：

1. `样片集`
2. `提交资料`
3. `制作状态`
4. `查看成片`

### 1.1 样片集

样片集是独立功能，用于展示成片效果和可参考模板方向。

样片集只回答：

- 有哪些好看的样片？
- 当前有哪些模板大类？
- 每个模板下有哪些子场景和样片组？
- 用户想参考哪一组样片？

样片集不承担：

- 上传参考照
- 生成三视图
- 选择正式生成参数
- 内部制作
- 交付管理

### 1.2 提交资料

提交资料固定为：

1. 注册或登录
2. 选择影像组合
3. 按人上传照片
4. 勾选影像风格或模板方向
5. 提交定制需求

硬规则：

- 未登录，不能提交定制订单。
- 未按人上传照片，不能提交定制需求。
- 每个人最多上传 9 张照片。
- 多人订单必须能分清每个人。

### 1.3 制作状态

客户只看到结果导向状态：

- `待提交`
- `已提交`
- `资料需补充`
- `制作中`
- `已完成`

### 1.4 查看成片

查看成片只展示正式结果。

必须做到：

- 成片来自正式交付结果。
- 支持查看、下载和反馈。
- 不展示过程稿、失败图、复核记录和工作人员备注。

## 2. 页面要求

### 2.1 首页

首页只回答两个问题：

- 当前订单到哪一步
- 下一步需要客户做什么

推荐布局：

1. 顶部：产品名、当前项目、服务状态
2. 样片集：模板大类、子场景、大图样片、缩略图
3. 提交资料：选择组合、按人上传照片、勾选风格
4. 制作状态：是否已提交、是否需要补充、是否完成
5. 查看成片：正式结果和下载入口

### 2.2 提交资料页

必须做到：

- 未登录时提示先登录
- 支持选择影像组合
- 支持按人上传照片
- 支持勾选风格或模板方向
- 主按钮统一为 `提交定制需求`

### 2.3 样片集页

样片集页只用于选模板和看样片，不承担最终交付。

必须做到：

- 大图优先
- 上方先选模板大类，并且平铺展示，不做横向滚动
- 下方左侧列子场景，右侧显示大图和缩略图
- 左侧子场景列表和右侧大图区域在桌面端上下齐平
- 样片图片优先看清人脸，不使用只能看到局部身体的横条裁切
- 能快速切换婚纱照、情侣照、闺蜜照、10 岁照、女生写真、夕阳红
- 样片集和正式结果区分开

### 2.4 后台制作平台

后台制作平台只给管理员和工作人员使用。

必须做到：

- 后台入口独立登录
- 可查看订单列表
- 可审核资料
- 可上传成果
- 可通知客户
- 内部制作状态不原样透传给客户

### 2.5 客户结果页

客户结果页只展示正式可用结果。

必须做到：

- 正式图片来自 `final_4k/`
- 样片、失败图、被替换旧图默认不进交付页
- 交付页只保留一个主动作：下载或确认交付

## 3. 状态门禁

### 3.1 账号门禁

客户平台只认以下状态：

- `待提交`
- `已提交`
- `资料需补充`
- `制作中`
- `已完成`

硬规则：

- 未注册或未登录，不能提交定制订单。
- 未登录，不能查看个人成片。
- 客户账号和后台管理员账号分开。

### 3.2 后台制作门禁

后台制作只在资料审核通过后启动。

当前技术口径：

- 正式批量阶段允许直接输出 4K 成片候选
- 批次复核直接看 4K 结果
- `final_4k/` 负责承接复核通过后的正式交付版本

后台状态建议：

- `新订单`
- `待审核`
- `需补传`
- `待制作`
- `制作中`
- `待复核`
- `待上传成果`
- `待通知客户`
- `已交付`

## 4. 部署门禁

### 4.1 启动前检查

- 工作区：`/Users/dy3000/code/pic`
- 依赖完整：`npm install` 已完成或 `node_modules` 存在
- 当前正式端口：`9999`
- 当前正式服务：`server/index.mjs`
- 当前正式成片目录：`final_4k/`

### 4.2 启动命令

```bash
node server/index.mjs --host 0.0.0.0 --port 9999
```

说明：

- 这是当前唯一正式启动命令
- 旧的 `identity-workflow` 独立启动方式不再作为现行部署口径

### 4.3 健康检查

```bash
curl -fsS http://127.0.0.1:9999/api/health
curl -fsS http://SERVER_IP:9999/api/health
```

通过标准：

- 返回 `ok: true`
- 服务名正确
- 页面可正常加载主界面

### 4.4 页面检查

部署前至少检查：

1. 首页能打开
2. 客户平台默认可见，并且只展示样片集、提交资料、制作状态、查看成片
3. 后台制作平台入口需要管理员登录
4. 没有页面级横向滚动
5. 按钮、标题、卡片不松散

## 5. 目录与交付门禁

### 5.1 正式交付目录

正式交付只认：

```text
final_4k/
```

### 5.2 历史兼容目录

历史重复目录是：

```text
成片/
```

它当前只允许：

- 保留历史数据
- 作为清理前的兼容痕迹

它当前不允许：

- 再作为正式交付目录
- 再作为脚本默认同步目标
- 再作为正式文档里的交付口径

1. 客户只记住 `9999`
2. 客户平台和后台制作平台边界清楚
3. 客户平台按注册用户提交资料和查看成片
4. 后台制作平台按管理员登录、审核资料、上传成果执行
5. 正式成片只认 `final_4k/`
6. 启动命令、健康检查、README、规格文档一致
