# 菜单竞品差距分析

日期：2026-05-02
模式：`doc + code`
工作区：`/Users/dy3000/code/pic`

## 1. 当前固定口径

一级菜单固定为 3 个：

1. `拍摄定制`
2. `图片创作`
3. `我的`

硬规则：

- 不新增第 4 个一级菜单。
- 不恢复 `连接设置`、`测试连接`。
- `final_4k/` 仍然是唯一正式成片目录，`我的作品` 只管生成历史和删除恢复。
- 新增二级菜单必须能落到真实界面，不做只有名字的空入口。

## 2. 竞品比我们多的常见能力

| 竞品能力 | 典型表达 | 对我们的判断 |
| --- | --- | --- |
| 个人模型 / AI Model | `My AI Model`、`Personal Model` | 对应我们的 `身份确认`，暂不新增一级菜单 |
| 照片包 / 场景包 | `Photo Packs`、`Style Packs` | 已对应 `场景套餐`，后续加强 |
| 编辑工具 | 换背景、换装、增强、修复 | 适合放进 `图片创作` 二级菜单 |
| 成片与下载 | Favorites、Downloads、HD、Ultra HD | 后续做作品管理增强，当前不和正式成片目录重复 |
| 团队 / 企业 | Team、Business、Brand | 暂不进当前客户前台 |
| 视频 | AI Video | 后续能力，不进当前试用版 |
| 隐私删除 | Delete model、Delete data | 后续应放上传和账户规则里，不做成连接设置 |

## 3. 本次试用版菜单

保持 3 个一级菜单，把作品、注册登录和我的积分统一放到 `我的`：

1. `拍摄定制`
   - `样片集`
   - `生图`
   - `身份确认`
   - `选模板`
   - `生成`
2. `图片创作`
   - `模板库`
   - `智能生图`
   - `换背景`
   - `换装`
   - `高清增强`
   - `局部修复`
3. `我的`
   - `全部作品`
   - `已删除`
   - `注册`
   - `登录`
   - `我的积分`
   - `充值中心`
   - `充值订单`
   - `积分流水`

## 4. 试用原则

- `图片创作` 的 4 个新增入口都进入同一个智能生图/编辑界面。
- 进入 `换背景`、`换装`、`高清增强`、`局部修复` 时，界面标题、提示词占位和素材入口文案要跟功能名一致。
- 如果没有上传原图，也要能看到清楚的起手提示。
- `我的积分` 必须保留 `充值订单` 和 `积分流水` 两个独立入口，不再用模糊的 `最近记录`。
- `我的作品` 当前不扩展 `成片`、`下载记录`，避免和 `拍摄定制 -> 成片交付`、`final_4k/` 重复。

## 5. 暂不做

- 不做独立 `AI 模型` 一级菜单。
- 不做团队/企业管理。
- 不做 AI 视频。
- 不做真实支付以外的新付费层级。
- 不把隐私、删除和上传规范做成分散菜单；后续应写进上传参考照和账户规则。

## 6. 验证标准

- 首页仍只有 3 个一级菜单。
- `图片创作` 下能看到 6 个二级入口。
- 点击新增编辑入口后，主界面标题和提示词引导与入口名称一致。
- `我的` 下能看到 `我的积分`、`充值订单`、`积分流水` 独立入口。
- 页面不出现 `连接设置`、`测试连接`。
- 通过 `npm run lint:strict`、`npm test`、`npm run typecheck`、`npm run verify:visual`。
