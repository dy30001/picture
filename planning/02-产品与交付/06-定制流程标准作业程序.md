# 定制流程标准作业程序

日期： 2026-05-02
模式： doc
工作区： `/Users/dy3000/code/pic`

## 0. 当前结论

当前“个人AI摄影棚”技术路线是通过的，但前提是严格按以下主线执行：

```text
选场景 -> 上传参考照 -> 确认像本人 -> 确认样片 -> 直接生成 4K 批次 -> 批次复核 -> 状态转正 -> 正式交付
```

不能跳过身份确认门禁，也不能把样片直接当成正式成片交付。

当前新增硬规则：

- 批量生成阶段允许直接产出 4K 成片候选。
- 批次复核直接基于 4K 结果进行。
- `转正` 指的是状态转正，不再默认代表“先生成中间尺寸，再二次转成 4K”。

## 1. 适用范围

这份标准流程适用于以下定制型项目：

- 婚纱旅拍
- 情侣写真
- 闺蜜写真
- 儿童写真
- 地标人像
- 旅行人像

它不解决“开放式自由生图”场景，只解决“真人参考照驱动、需要交付结果”的定制项目。

## 2. 一句话主流程

客户视角只需要理解 2 个功能和 4 个生图步骤：

1. 样片集：先看样片、场景和参考方向。
2. 生图第 1 步：传三张图。
3. 生图第 2 步：确认像本人。
4. 生图第 3 步：选模板作为生成参考。
5. 生图第 4 步：正式生成。

样片集内部口径：先选大场景，再看样片组矩阵，当前组内照片同屏预览；不混入上传、身份确认和模板选择。

内部视角则必须拆成 8 个执行阶段：

1. 项目建档
2. 参考图收集与初筛
3. 身份确认
4. 样片试拍与方向确认
5. 直接生成 4K 批次
6. 批次复核
7. 返修与状态转正
8. 正式交付

## 3. 客户版流程

### 3.1 选场景

客户先选当前要做的场景包或定制方向，例如：

- 婚纱旅拍
- 地标旅拍
- 情侣日常
- 闺蜜纪念
- 儿童成长

本阶段输入：

- 场景需求
- 风格偏好
- 是否为单人、双人或多人

本阶段输出：

- `sceneId`
- 当前定制方向
- 当前项目草稿

### 3.2 上传参考照

客户上传本人参考照。系统不要求客户理解模型，只要求客户提供足够清晰、足够稳定的照片。

最低要求：

- 脸清楚
- 遮挡少
- 至少能看清五官和脸型
- 双人项目要能分清双方身份

本阶段输出：

- 原始参考图集合
- 可进入初筛的项目记录

### 3.3 身份确认

这是整个流程最关键的第一道门禁。

内部先生成：

- 女方或单人身份确认图
- 男方身份确认图
- 双人比例图

客户只需要判断一件事：

- 像不像本人

通过标准：

- 五官稳定
- 身高关系合理
- 没有明显 AI 结构错误

未通过：

- 只能返修身份确认图
- 不能进入样片阶段

### 3.4 确认样片

身份确认通过后，不直接批量跑全量，而是先出少量样片定方向。

样片负责确认：

- 风格对不对
- 服装对不对
- 氛围对不对
- 景区方向对不对
- 人脸是否还能保持

样片不负责：

- 最终交付
- 完整数量
- 最终高清成片

### 3.5 批量生成

客户确认样片方向后，才进入正式批量生成。

批量生成默认按“景区或主题批次”推进，而不是一次性全量完成。

当前标准口径：

- 批量阶段直接生成 4K 成片候选。
- 不再把“先出中间尺寸，再统一转 4K”作为默认流程。

这样做的原因是：

- 脸不像时能及时止损
- 服装或景区跑偏时能及时回调
- 返修成本比全量推倒低

### 3.6 成片复核

批量生成后，先由内部复核，不直接对客户暴露全部结果。

当前复核对象默认就是 4K 候选图，而不是中间尺寸样图。

复核内容包括：

- 人脸相似度
- 手部和肢体结构
- 婚纱与服装完整度
- 构图是否符合镜头意图
- 地标和场景是否清晰可识别
- 是否有水印、乱码、错误文字

### 3.7 正式交付

只有通过复核、被转正的图片，才进入正式交付阶段。

当前唯一正式交付目录是：

```text
final_4k/
```

客户最后只看正式成片，不看失败图、旧图和被淘汰样片。

## 4. 内部执行版 SOP

### 4.1 项目建档

创建项目时至少建立以下信息：

- `projectId`
- `sceneId`
- `peopleType`
- `referenceCount`
- `identityStatus`
- `sampleStatus`
- `deliveryStatus`

本阶段完成标准：

- 项目可被追踪
- 当前定制方向明确
- 参考图入口可用

### 4.2 参考图初筛

内部从客户上传图里筛出：

- 女方脸部参考
- 女方体态参考
- 男方脸部参考
- 男方体态参考
- 双人关系参考

筛选原则：

- 正脸优先
- 光线均匀
- 遮挡少
- 表情自然
- 双人项目优先保留能看出关系感的图

本阶段输出：

- 选中参考图集
- 失效图记录
- 身份基准输入集

### 4.3 身份确认

内部生成以下身份基准：

- 女方正面 / 侧面 / 背面
- 男方正面 / 侧面 / 背面
- 双人比例图

本阶段目标不是好看，而是“身份稳定、比例稳定、关系稳定”。

本阶段输出：

- `identityStatus = 待确认` 或 `需返修`
- 身份确认预览
- 双人比例图

### 4.4 样片试拍

身份确认通过后，按当前场景方向生成少量样片。

样片阶段建议控制在：

- 每个方向 1 到 3 张
- 或每个景区先出 1 张试拍图

样片阶段要回答的问题：

- 这套风格对不对
- 这套服装对不对
- 这套景区对不对
- 这张脸还能不能认出来

本阶段输出：

- 样片图
- 样片联系表
- 客户确认意见

### 4.5 批量生成

样片确认后，进入正式批量生成。

默认方式：

- 按景区推进
- 每个景区一批
- 每批直接生成 4K 候选图
- 批次过关后再进入下一批

本阶段输出：

- `generated/` 或正式批次目录中的 4K 候选图
- `review/contact_sheets/` 联系表
- 对应批次记录

### 4.6 批次复核

每一批至少要完成：

1. 文件完整性检查
2. 快速视觉筛查
3. 单张质量判断
4. 问题图标记
5. 是否进入返修队列

当前默认前提：

- 复核基于 4K 候选图进行
- 联系表只是复核视图，不是单独的中间交付尺寸

本阶段输出：

- `待重生`
- `已入围`
- `已淘汰`
- `已转正`

### 4.7 返修与转正

返修只针对问题图，不回退已经合格的图。

转正规则：

- 只有通过批次复核的图
- 才允许进入 `final_4k/`
- 每个镜头编号只保留一个正式版本

说明：

- 这里的 `转正` 是状态动作和目录动作。
- 如果批次已经直接生成 4K，那么转正时默认不再需要额外做一次尺寸转换。

本阶段输出：

- 正式 4K 成片
- 历史版本保留
- 返修记录可追踪

### 4.8 交付确认

交付前只做 3 件事：

1. 核数量
2. 核目录
3. 核最终客户可见内容

交付阶段页面只保留一个主动作：

- 下载
- 或确认交付

## 5. 三个硬门禁

### 5.1 身份门禁

身份未确认前：

- 不准进入正式样片确认
- 不准进入批量生成

### 5.2 样片门禁

样片方向未确认前：

- 不准启动整批成片
- 不准承诺最终交付数量

### 5.3 交付门禁

图片未进入 `final_4k/` 前：

- 不算正式成片
- 不进入交付页
- 不作为客户最终下载结果

补充规则：

- 批次可以直接生成 4K，但只有复核通过并进入 `final_4k/` 后，才算正式交付。

## 7. 页面模块拆分建议

前台主流程建议固定为 5 个一级面板：

1. 场景套餐
2. 身份确认
3. 样片确认
4. 批量成片
5. 成片交付

每个面板的重点分别是：

- 场景套餐：选方向
- 身份确认：看是否像本人
- 样片确认：先看少量样片定风格
- 批量成片：看当前批次进度和复核状态
- 成片交付：只放正式可用图

## 8. 最小可用版实施顺序

第一阶段先做通主线，不追求一次把所有后台做满。

### 第一阶段

- 跑通场景选择
- 跑通参考照上传
- 跑通身份确认
- 跑通样片确认
- 跑通交付目录展示

### 第二阶段

- 增加样片状态管理
- 增加批次复核状态
- 增加返修队列
- 增加转正记录

### 第三阶段

- 增加项目级历史记录
- 增加客户项目切换
- 增加下载包和交付确认
- 增加更完整的后台追踪

## 9. 当前最大风险

这条路线虽然通过，但现在有 4 个主要风险还没有完全产品化：

1. 项目状态还不够持久化
2. 样片管理还偏工作台，不是完整模块
3. 返修到转正的动作链还不够正式
4. 客户界面和内部执行状态还没有完全打通

## 10. 完成标准

这份标准流程成立，至少要满足以下条件：

1. 客户能理解自己当前做到哪一步
2. 内部知道每一步的输入、输出和门禁
3. 身份确认、样片、成片、交付边界清楚
4. `final_4k/` 仍作为唯一正式交付目录
5. 批次复核默认可以直接基于 4K 结果进行
6. 后续页面、状态机和实现计划都能以本文件为唯一主线入口
