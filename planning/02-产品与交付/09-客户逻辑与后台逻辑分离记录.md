# 客户逻辑与后台逻辑分离记录

日期：2026-05-04
工作区：`/Users/dy3000/code/pic`

## 1. 客户逻辑

当前模式已固化：客户前台只做客户看得懂的服务入口，不展示生产过程。

前台结构固定为：

- `首页`：讲生活愿景和服务价值
- `拍摄定制`：客户提交定制资料
- `图片创作`：客户自己生成图片
- `我的作品`：客户查看历史结果
- `登录`：客户账号入口
- `我的积分`：客户积分和充值

后台制作平台不在客户前台展示；后续单独开内网端口给管理员和工作人员使用。

客户只用四个功能：

1. `样片集`
2. `提交资料`
3. `制作状态`
4. `查看成片`

客户只做这些事：

- 看样片集，先选模板大类，再看子场景和具体样片
- 选择拍摄组合和风格方向
- 按人上传照片，每人最多 9 张
- 提交定制需求
- 查看提交内容、当前状态和正式成片

## 2. 后台逻辑

后台只给管理员和工作人员用：

1. `管理员登录`
2. `接收订单`
3. `审核资料`
4. `制作成片`
5. `上传成果`
6. `通知客户`
7. `完成交付`

后台生产成果页后续单独开内网端口：管理员登录后可以看订单、改状态、上传成果；客户前台不处理这些动作，也不展示后台入口。

## 3. 不给客户看的内容

- 三视图
- 人物确认图
- 像不像审核
- 身份确认
- 内部复核
- 制作过程稿
- 工作人员备注

## 4. 宣传语言

宣传语言围绕一句主张：

> 让向往的生活，先在照片里发生。

客户能感知的价值：

- 旧照修复
- 亲人同框
- 爱人补拍
- 正式成片
- 把遗憾补成可以保存的作品

宣传页讲生活和结果，不讲操作教程；流程说明只放在提交、状态或后台页面。

## 5. 本次完成标准

- 客户二级菜单只显示：`样片集 / 提交资料 / 制作状态 / 查看成片`
- `样片集` 只显示模板大类、子场景/样片组、大图、缩略图和放大预览
- `提交资料` 只保留组合、风格、按人照片、备注和提交
- `制作状态` 只显示客户提交了什么、当前到哪一步、是否需要补资料
- 正式成果只在 `查看成片` 展示
- 后台成果上传只在内部后台平台处理，客户前台不展示入口。
