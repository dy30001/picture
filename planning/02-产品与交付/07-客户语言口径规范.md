# 客户语言口径规范

日期： 2026-05-03
模式： `doc`
工作区： `/Users/dy3000/code/pic`

## 0. 目的

这份规范只管一件事：客户界面和对外说明里，只保留客户能直接理解的词，不再混用内部逻辑词。

## 1. 总原则

- 只说动作、结果和状态。
- 只保留客户能扫一眼就懂的词。
- 每个功能独立展示，不把多个功能混成一页。
- 客户界面不解释系统，不讲内部字段名。
- 对外销售界面不写“先做 A、再做 B、等待 C”这种说教式流程话术。
- 首页、主按钮、卡片标题和主区说明优先写客户价值：修复、重逢、补拍、成片、交付。
- 流程顺序只能放在必要的状态或后台说明里，不能当作对外卖点。

## 2. 不出现在客户界面的词

- `账户权益`
- `权益总览`
- `余额`
- `三视图`
- `人物确认图`
- `像不像审核`
- `后台上传`
- `身份确认`
- `数字底片`
- `加料项`
- `继续`
- `第 N 步`
- `生成成片`
- `主提示词`
- `加料`
- `先……再……`
- `先做……再做……`
- `等待……`
- `第 N 步`
- `步骤 N`
- `下一步`
- `精选优先`
- `优先套餐`
- `第一组`

## 3. 推荐写法

| 内部词 | 客户写法 |
| --- | --- |
| `账户权益` | `我的` / `我的积分` |
| `权益总览` | `我的积分` |
| `余额` | `可用积分` |
| `三视图` | `提交资料` / `等待制作` |
| `人物确认图` | `制作中` |
| `像不像审核` | `资料已收到，正在制作` |
| `后台上传` | `成片已完成` |
| `身份确认` | `提交资料` / `制作状态` |
| `数字底片` | `参考照` |
| `加料项` | `补充效果` |
| `继续` | 具体动作，例如 `查看成片`、`提交资料`、`买积分` |
| `第 N 步` | 直接写动作或状态，例如 `提交资料`、`制作中`、`成片已完成` |
| `生成成片` | `开拍成片` / `生成` |
| `主提示词` | `画面描述` / `想拍什么` |
| `加料` | `补充效果` |

## 4. 对外销售话术规则

### 4.1 禁止说教式流程

不要在客户主界面写：

- `先看方向，再提交资料`
- `先上传照片，再写提示词`
- `先登录，再提交资料`
- `等待正式成片`
- `第 1 步 / 第 2 步`

推荐写：

| 不推荐 | 推荐 |
| --- | --- |
| `先看方向，再提交资料` | `喜欢的风格一眼选定` |
| `先上传照片，再写提示词` | `上传照片，描述想要的画面` |
| `先登录，再提交资料` | `登录后可提交资料` |
| `等待正式成片` | `正式作品交付` |
| `第 1 步 上传照片` | `上传照片` |
| `精选优先` | `只看精选` |
| `优先套餐` | `推荐套餐` |
| `第一组样片` | `对应样片组` |

### 4.2 首页和销售页口径

- 首页讲客户愿景，不讲操作教程。
- 首页主标题固定用品牌主张：`让向往的生活，先在照片里发生`。
- 首页说明优先表达结果：`旧照修复`、`亲人同框`、`爱人补拍`、`正式成片`。
- 首页卡片只写客户能买到的服务和结果，不写“先、再、然后、等待”。
- 对外销售页不是内部 SOP，不展示后台生产细节。

## 5. 现在统一口径

- 一级菜单：`首页` / `拍摄定制` / `图片创作` / `我的作品` / `登录` / `我的积分`
- 后台制作平台不在客户前台展示；后续单独开内网端口。
- 品牌主张：`让向往的生活，先在照片里发生`
- 首页：`旧照修复`、`亲人同框`、`爱人补拍`、`正式成片`
- 拍摄定制：`看效果`、`提交资料`、`制作状态`、`查看成片`
- 登录：`邮箱登录`、`创建账号`
- 积分页只说 `我的积分`、`可用积分`、`买积分`、`充值记录`、`积分明细`、`出图成功才扣积分`
- 买积分前必须登录；未登录时只提示 `登录后可购买`
- 可选编辑统一说 `补充效果`
- 智能生图主区优先说 `上传照片`、`画面描述`、`开拍`
- `换背景`、`换装`、`高清增强`、`局部修复` 默认归到 `补充效果`，作为画面描述的补充
- `提示词怎么写`、`模板怎么借`、`写法参考` 默认放在右侧下方，不写成主区大段解释
- `图片创作` 首页主区统一写 `趣味生图`、`套图写真`
- `模板库` 只作为现阶段路由名、次级入口或筛选口径，不再当客户首页主标题
- 客户提交定制时统一写 `提交定制需求`
- 客户制作状态只写 `已提交`、`资料需补充`、`制作中`、`已完成`
- 后台工作人员动作只出现在后台制作平台，不写进客户前台
