# 图像生成技能更新交接

## 一句话目标
把同事电脑上的 `imagegen` skill 更新到能正确使用 OpenAI `Responses API + image_generation tool` 生成/编辑图片；`gpt-image-2` 不能填到 `responses.create(model=...)`，要通过 `tools: [{"type": "image_generation"}]` 触发图像工具。

## 模式
- `doc mode`：先改 `SKILL.md` 和 reference 文档，让 Codex 会判断什么时候走 Responses 图像工具。
- `code mode`：如果 skill 里有 `scripts/image_gen.py` 这类 CLI fallback，再改脚本实现和测试。

## 工作区
在同事电脑上定位实际 skill 目录，常见路径：

```bash
echo "${CODEX_HOME:-$HOME/.codex}"
find "${CODEX_HOME:-$HOME/.codex}/skills" -path "*/imagegen/SKILL.md" -print
```

优先修改实际生效的那个 `imagegen` skill，不要改项目仓库里的普通文档当作替代。

## 官方口径
- Responses API 调图像：`model` 使用主线模型，例如 `gpt-5.5` 或账号可用的 `gpt-5`；图像能力通过 `tools: [{"type": "image_generation"}]` 挂载。
- 图像结果在 `response.output` 里，筛选 `output.type == "image_generation_call"`，从 `output.result` 取 base64。
- 如果要强制调用图像工具，设置 `tool_choice={"type": "image_generation"}`。
- `gpt-image-2` 是图像工具背后的 GPT Image 模型之一，但不是 Responses API 的 `model` 字段合法值。
- `image_generation` tool 常用参数放在 tool 对象里：`size`、`quality`、`output_format`、`output_compression`、`background`、`partial_images`、`action`。
- `gpt-image-2` 目前不支持 `background: "transparent"`；透明图不要直接这样调，继续用 chroma-key 后处理，或经用户确认后走可用的透明 fallback。

官方文档：
- https://developers.openai.com/api/docs/guides/tools-image-generation
- https://platform.openai.com/docs/guides/image-generation?api=responses
- https://platform.openai.com/docs/api-reference/responses/create

## 非目标
- 不要把 `responses.create(model="gpt-image-2")` 当作修复。
- 不要为了这件事重写整个 skill。
- 不要新增一次性脚本绕过 skill。
- 不要要求用户把 API key 发到聊天里；只要求用户在本机设置 `OPENAI_API_KEY`。
- 不要宣称内置 Codex `image_gen` 工具需要 API key；只有 CLI/API fallback 需要。

## 需要影响的文件
按同事电脑实际目录为准，通常是：

```text
<imagegen>/SKILL.md
<imagegen>/references/cli.md
<imagegen>/references/image-api.md
<imagegen>/scripts/image_gen.py
```

如果没有 CLI 脚本，只改 `SKILL.md` 和 reference 文档即可。

## SKILL.md 必改内容
在 top-level modes / fallback CLI mode 附近加入这段规则，按原文结构压缩合并：

````markdown
### Responses image_generation 工具规则

For API/CLI fallback with `gpt-image-2`, use the Responses API hosted `image_generation` tool.
Do not set `model: "gpt-image-2"` in `responses.create`; GPT Image models are used by the hosted image tool and are not valid mainline `model` values for Responses API.
Use a text-capable mainline model such as `gpt-5.5` or an available `gpt-5` family model:

```python
response = client.responses.create(
    model="gpt-5.5",
    input=prompt,
    tools=[{
        "type": "image_generation",
        "size": size,
        "quality": quality,
        "output_format": output_format,
    }],
    tool_choice={"type": "image_generation"},
)
image_data = [
    output.result
    for output in response.output
    if output.type == "image_generation_call"
]
```

Tool options belong inside the `image_generation` tool object: `size`, `quality`, `output_format`, `output_compression`, `background`, and `action`.
For `gpt-image-2`, do not use `background: "transparent"`; it fails. Use the built-in chroma-key workflow first, or ask before a true-transparency fallback.
````

注意：上面用了四反引号包 Markdown，实际粘贴到 `SKILL.md` 时只保留内部正文即可。

## CLI 脚本实现要求
如果存在 `scripts/image_gen.py`，把 `gpt-image-2` 的生成/编辑默认路径从 `client.images.generate/edit(...)` 改成 `client.responses.create(...) + image_generation tool`。

建议实现方式：

1. 保留原来的 prompt augmentation、输出路径、base64 解码写文件、downscale、dry-run、参数校验。
2. 新增 `--response-model`，默认 `gpt-5.5`；如果账号没有该模型，允许调用者传 `--response-model gpt-5`。
3. `--model gpt-image-2` 时走 Responses tool；不要把 `gpt-image-2` 传给 `responses.create(model=...)`。
4. `--model gpt-image-1.5 --background transparent` 等透明 fallback 可以保留旧 Images API 路径，但必须是用户明确选择。
5. Responses tool 可能没有 `n` 参数；如果 CLI 还支持 `--n > 1`，用循环发多次 Responses 请求，或明确限制并报错。不要把未知参数塞进 tool。
6. 编辑本地图片时，用 base64 data URL 作为 `input_image`，示例：

```python
def _image_to_data_url(path: Path) -> str:
    mime = "image/png"
    if path.suffix.lower() in {".jpg", ".jpeg"}:
        mime = "image/jpeg"
    elif path.suffix.lower() == ".webp":
        mime = "image/webp"
    b64 = base64.b64encode(path.read_bytes()).decode("ascii")
    return f"data:{mime};base64,{b64}"

input_payload = [{
    "role": "user",
    "content": [
        {"type": "input_text", "text": prompt},
        *[
            {"type": "input_image", "image_url": _image_to_data_url(path)}
            for path in image_paths
        ],
    ],
}]

tool = {
    "type": "image_generation",
    "size": args.size,
    "quality": args.quality,
    "output_format": output_format,
}
if args.output_compression is not None:
    tool["output_compression"] = args.output_compression
if args.background is not None:
    tool["background"] = args.background
tool["action"] = "edit"  # generate/edit/auto,按命令设置

response = client.responses.create(
    model=args.response_model,
    input=input_payload,
    tools=[tool],
    tool_choice={"type": "image_generation"},
)

images = [
    output.result
    for output in response.output
    if output.type == "image_generation_call"
]
```

生成纯文本 prompt 时，`input` 可以直接用字符串，也可以使用同样的 message content 结构；为了统一，脚本里可以都用 message content 结构。

## 文档同步要求
在 `references/cli.md` / `references/image-api.md` 里同步说明：

- `gpt-image-2` 的推荐 CLI/API fallback 是 Responses API hosted `image_generation` tool。
- `--model gpt-image-2` 只是用户侧意图，不是 `responses.create(model=...)` 的值。
- `--response-model` 才是 Responses API 主线模型，默认 `gpt-5.5`，可改 `gpt-5`。
- dry-run 输出要显示 `endpoint: /v1/responses`，并展示 `tools: [{"type": "image_generation", ...}]`。

## 验证方案
必须至少做这些检查：

```bash
python -m py_compile scripts/image_gen.py
python scripts/image_gen.py generate --prompt "Draw a small red square on white background" --model gpt-image-2 --dry-run
python scripts/image_gen.py edit --prompt "Edit the image to add a blue border" --model gpt-image-2 --image <本地测试图> --dry-run
```

验收点：
- dry-run 里是 `/v1/responses`，不是 `/v1/images/generations`。
- dry-run 里 `model` 是 `gpt-5.5` 或 `--response-model` 指定值，不是 `gpt-image-2`。
- dry-run 里有 `tools: [{"type": "image_generation", ...}]`。
- `background: "transparent"` 搭配 `gpt-image-2` 会被阻止或引导到 chroma-key/确认流程。
- 如果设置了 `OPENAI_API_KEY` 并允许真调用，再跑一个最小实测，确认生成文件能写出且能被图片工具打开。
