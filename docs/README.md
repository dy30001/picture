| 位置 | 作用 | 是否为源文档 |

## 4. 更新规则

- 新增长期规则、命名口径、入口索引：放 `docs/`。
- 新增工作流规格、计划、交接、执行报告：放 `planning/`。
- 过程文档命名统一用 `主题-执行计划.md`、`主题-执行报告.md`、`主题-YYYYMMDDHHMMSS-执行报告.md`、`主题.csv`。
- 新增客户可见的产品说明、启动说明：回写 `README.md`。
- 新增 `planning/` 主线文档后，要同步更新 `planning/README.md`。
- 新增长期入口或规范文档后，要同步更新本文件。

## 5. 当前硬规则

- 内部统一服务端口只认 `http://<host>:9999`，客户前台只展示正式域名和品牌。
- 默认主服务只认 `server/index.mjs`。
- 客户前端默认已连好，不展示 `连接设置`、`测试连接` 这类技术入口。
- 客户前台不展示内部端口号，也不放备案号。
- 当前客户前台设计采用 5 个业务板块：`拍摄定制` / `图片创作` / `我的作品` / `登录` / `我的积分`。
- 5 个业务板块不是界面排版规则；前端可响应式呈现。后续如果业务设计要调整板块数量、命名或顺序，先改文档确认，再改代码。
- 图片创作现阶段路由仍为 `模板库`、`智能生图`；客户前台主区推荐按 `趣味生图`、`套图写真` 双区块表达，编辑类能力继续放在智能生图页面内。
- `我的积分` 是客户前台积分入口，不再使用旧口径 `账户权益`。
- 香港公网默认按“代码 push + 服务器 pull，图片直拷”更新，不把大批图片上传混进代码发布链路。
- 外部模板图默认按“本地预览图优先，外链只作来源元数据”处理。
- 那 `17` 个本地项不算模板，统一按本地样片/封面素材口径描述。
- `final_4k/` 是唯一正式成片目录。
- `成片/` 只保留历史重复数据，不再作为正式交付口径。
- 人像类流程保持 `先确认像本人，再出模板样片`。
- 正式 icon 只认 Air 原件，不能按截图重画，也不能让品牌资产脚本重新生成。
- 历史计划或执行报告里如果还写着 `assets:brand` 会生成图标，一律按过时口径处理。

## 6. 不要这样做

- 不要再把新的过程文档堆到仓库根目录。
- 不要直接编辑 `dist/` 里的文档副本。
- 不要只写 `*-执行报告.md` 而不把长期结论沉淀到规范或索引里。
